# 🚀 ProdHunt India — Job Tracker for Freshers

A free, self-hosted job board that scrapes **Product & Growth roles in India** daily and displays them on a beautiful website — automatically.

**Target roles:** Product Intern · Growth Intern · Associate Product Manager · Product Manager  
**Experience:** 0–2 years / Freshers only  
**Location:** India (all cities + Remote)  
**Sites scraped:** hiring.cafe · LinkedIn · Naukri · Wellfound · Internshala

---

## 🗂️ File Structure

```
job-tracker/
├── scraper.py                    ← Python scraper (runs daily)
├── index.html                    ← The website (reads jobs.json)
├── data/
│   └── jobs.json                 ← Auto-generated by scraper
├── .github/
│   └── workflows/
│       └── scrape.yml            ← GitHub Actions (daily automation)
└── README.md
```

## ⚡ Setup in 10 Minutes (Step by Step)

### Step 1 — Create a GitHub Account
Go to [github.com](https://github.com) and sign up (free).

---

### Step 2 — Create a New Repository
1. Click the **+** icon → **New repository**
2. Name it: `job-tracker`
3. Set it to **Public** (required for free GitHub Pages hosting)
4. Click **Create repository**

---

### Step 3 — Upload the Files
Upload these 3 files to your repo root:
- `scraper.py`
- `index.html`
- `README.md`

Also create the folder path `.github/workflows/` and upload `scrape.yml` there.

**Easy way:** Click **Add file → Upload files** in the GitHub UI.

---

### Step 4 — Create the `data/` folder
GitHub doesn't allow empty folders. Do this:
1. Click **Add file → Create new file**
2. In the filename box type: `data/.gitkeep`
3. Click **Commit new file**

---

### Step 5 — Enable GitHub Pages (Free Hosting)
1. Go to your repo → **Settings** → **Pages** (left sidebar)
2. Under **Source** → select **Deploy from a branch**
3. Select branch: `main`, folder: `/ (root)`
4. Click **Save**

Your site will be live at:  
`https://YOUR_USERNAME.github.io/job-tracker`

(Takes ~2 minutes to go live the first time)

---

### Step 6 — Run the Scraper Manually (First Time)
1. Go to your repo → **Actions** tab
2. Click **Daily Job Scraper** in the left sidebar
3. Click **Run workflow** → **Run workflow**
4. Wait ~2 minutes for it to finish
5. Check your repo — `data/jobs.json` should now appear with real jobs!
6. Refresh your GitHub Pages site — jobs will show up 🎉

---

### Step 7 — Automation is Ready!
The scraper now runs **automatically every day at 7:00 AM IST**.  
No manual work needed after this point.

## 🛠️ Customization

| What to change | Where |
|---|---|
| Add/remove job roles | `TARGET_ROLES` list in `scraper.py` |
| Change scrape time | `cron:` line in `scrape.yml` (UTC time) |
| Change site colors | CSS variables at top of `index.html` |
| Add more cities | `INDIA_CITIES` list in `scraper.py` |
| Change experience filter | `MAX_EXP_YEARS` in `scraper.py` |

## ⚠️ Important Notes

- **LinkedIn** may block scraping — if it stops working, the other 4 sites will still run fine
- **Wellfound** requires Playwright (JS rendering) — handled automatically in GitHub Actions
- If a site structure changes, the scraper for that site may return 0 results — check Actions logs
- All data is stored in `data/jobs.json` — this is public since the repo is public

---

## 🐛 Troubleshooting

**Site shows "Demo mode":**  
→ The scraper hasn't run yet. Go to Actions tab and trigger it manually (Step 6).

**GitHub Actions failing:**  
→ Go to Actions tab → click the failed run → read the error log

**0 jobs from LinkedIn:**  
→ Normal — LinkedIn blocks scrapers periodically. Other sites will still work.

**Jobs not updating on site:**  
→ Hard refresh the page (Ctrl+Shift+R) — browser may be caching
